当前圆桌界面把"卡片摘要"和"shell PTY 流"塞在同一棵 DOM 树里相互争抢空间。所有视觉/渲染问题都是这个混合架构的衍生：

用户报告本质根因本次解法
#1 shell 压住底部按钮flex 高度计算 + z-index 在混合架构里争抢圆桌界面无 shell → 按钮永远在上
#2 Gemini shell 渲染怪父布局抖动触发 xterm.fit() 失稳shell 进入子 session 独立 view，不再被父布局抢空间
#5 卡死 3min 后才出逃生逃生按钮被沉浸 visibility 控制 + 时延门槛逃生按钮常驻 + 新增"进 shell"直接看真相
#6 模式切换渲染重复普通发言/群策群力切换 → DOM 重建 → xterm 重 fit删除"沉浸/调试"双视图开关，模式只剩输入策略
#7 摘要截断保尾去头slice(-2000) 截头保尾反转为 slice(0, 2000) + 截断提示
#8 普通→群策群力终端被压同 #6 根因同 #6 解法

四、xterm 容器单 viewer 切换流程

用户点 sidebar 子 session 行
→
从 terminalCache.get(sid)
取 container
→
if container.parentElement
则 detach
→
attach 到
#shell-view-mount
→
rAF + fitAddon.fit()
显示 panel

反向（离开子 session）：从 #shell-view-mount detach → 不重新 attach（暂存内存）。PTY 进程一直存活，xterm 实例从未销毁，ringBuffer 持续承接输出。下次 attach 时 xterm 自动续写补齐缺失帧。
