Prompt 注入架构优化

从"两套平行模板"到"基础规则 + 场景插件"

当前问题诊断

问题 1：两套 Rules 模板 80% 重复

general-roundtable-mode.js 和 research-mode.js 各自维护一份完整的 rules 模板。角色定义、语法说明(@debate/@summary)、协作礼仪几乎完全一样，只有数据获取段和少量措辞不同。改一处要同步改另一处。

问题 2：Orchestrator 硬编码"投研圆桌"

roundtable-orchestrator.js 的 turn prompt 全部写死 [投研圆桌 · 第N轮 · ...]，通用圆桌也会收到这个前缀 —— 场景无关的 orchestrator 和场景强耦合。

问题 3：Summary 输出格式绑死投研

Summary prompt 的输出建议写死了"仓位/止损/加仓/观察指标"、<<TITLE: 兆易创新-买入决策>> —— 通用圆桌讨论技术架构时也会收到这些投研专属指引。

问题 4：两套平行的文件管理函数

两个模块各自实现 writeXxxPromptFile / writeCovenantSnapshot / readCovenantSnapshot / cleanupXxxFiles —— 逻辑近乎一致，只是文件名前缀不同（-roundtable.md vs -research.md）。

场景注册表（核心数据结构）

新增：SCENE_REGISTRY 对象

一个场景 = 一组配置字段。新增场景只需往注册表加一条，零 UI 代码改动。

// core/roundtable-scenes.js (新文件)
+ const SCENE_REGISTRY = {
+ general: {
+ name: '通用圆桌',
+ icon: '🎯',
+ preset: SCENE_GENERAL, // ~5行场景专属指引
+ defaultCovenant: '', // 空
+ mcpConfig: null, // 无额外 MCP
+ summaryHints: '按讨论话题自适应', // summary 输出指引
+ summaryTitleTag: false, // 不要求 <<TITLE>>
+ dataPackEnabled: false, // 不自动拉数据
+ },
+ research: {
+ name: '投研圆桌',
+ icon: '📊',
+ preset: SCENE_RESEARCH, // ~40行投研指引
+ defaultCovenant: COVENANT_TEMPLATE,
+ mcpConfig: buildResearchMcpConfig,
+ summaryHints: '仓位/止损/加仓/观察指标',
+ summaryTitleTag: true, // 要求 <<TITLE>> 用于决策档案
+ dataPackEnabled: true, // fanout 时自动拉 LinDangAgent
+ },
+ // 未来扩展示例：
+ // codeReview: { name: '代码审查', preset: SCENE_CODE_REVIEW, ... },
+ // pptDesign: { name: 'PPT 讨论', preset: SCENE_PPT, ... },
+ };
字段用途通用圆桌投研圆桌
name Turn prompt 前缀 + UI 显示 通用圆桌 投研圆桌
preset 场景专属 prompt 片段（拼在 Base Rules 之后） ~5行（通用工具描述） ~40行（LinDangAgent + 数据优先级）
defaultCovenant 新建会议时的默认公约 空 投研公约 v1（投资风格/红线）
mcpConfig 额外 MCP server 配置 null arena-research server
summaryHints @summary 输出格式提示 按话题自适应 仓位/止损/加仓/观察指标
dataPackEnabled fanout 时是否自动拉数据包 false true（LinDangAgent）

优化总结

1
Rules 去重：两套 → 一套 Base Rules
抽出共享部分（角色 + 语法 + 礼仪 + 工具 + 留白）作为 BASE_ROUNDTABLE_RULES。统一包含 @<who> 私聊语法（投研之前没有但应该有）。改 rules 只改一处。
2
场景注册表驱动：新增场景零 UI 改动
每个场景只是一条 SCENE_REGISTRY 记录（name/preset/covenant/mcp/summaryHints）。header toggle 按钮从注册表动态生成，新增场景只需加一条记录。
3
Orchestrator 场景感知
构造函数接收 sceneName，turn prompt 前缀从 [投研圆桌·第N轮] 变为 [{sceneName}·第N轮]。summary 输出提示从 scene.summaryHints 读取。
4
文件管理统一
一套 writePromptFile / writeCovenantSnapshot / readCovenantSnapshot / cleanup 函数。文件名统一 {meetingId}-prompt.md / {meetingId}-covenant.md（不再按模式分命名）。
5
投研私聊补齐
当前投研 rules 缺 @<who> 私聊 语法说明（只有通用版有）。统一到 Base Rules 后两个场景都自动拥有私聊能力。
