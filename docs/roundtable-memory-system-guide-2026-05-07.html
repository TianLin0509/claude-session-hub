一、当前系统能力总览

你使用时会感受到的 5 件事

#能力体现
1 AI 会自己决定"该记了" 聊到你的稳定偏好/风格/习惯时，AI 自动调 memory_write 记到自己的记忆文件。不需要你说"记住"——但你说了会 100% 记（source=explicit）。
2 下次同 scene 圆桌，AI 自然引用上回的理解 通用的圆桌里 AI 记住了"你喜欢结论先行"→ 第二天再开通用圆桌 → AI 在回答里自然带出："上次你提过偏好直接给判断，我就不铺背景了。"——不是复读机，是自然延续。
3 每 3 轮后台 worker + DeepSeek 提炼三家共识 3 轮后 worker 自动 fork → DeepSeek 读取三家各自记忆 → 提炼出至少 2 家共现的稳定偏好 → 写入 _profile.md（上限 20 条）。下次新 meeting 开场自动注入给三家 AI。
4 每张 AI 卡片右上角可实时查看记忆 📒 N = 该 AI 个体记忆条数（实时更新），点击直接打开 .md 文件；📊 = 打开共识 _profile.md；📥 红点 = worker 发现漏记的候选项；🧠 = worker 健康状态灯。
5 通用 vs 投研互不污染 通用圆桌的记忆存在 general/ 下，投研的记忆在 research/ 下。通用圆桌学到的偏好不会影响投研圆桌的判断，反之亦然。

二、记忆归属关系（你最关心的核心）

归属的三层结构

记忆归属 = Identity（AI 身份）→ Scene（场景）→ File 具体文件： .arena/rooms/{scene}/memory/{aiKind}-{model}.md 实例： .arena/rooms/general/memory/claude-opus-4-7.md ← Opus 4.7 在通用圆桌的所有记忆 .arena/rooms/general/memory/claude-sonnet-4-6.md ← Sonnet 4.6 在通用圆桌的所有记忆 .arena/rooms/general/memory/deepseek-v4-pro.md ← DeepSeek v4-pro 在通用圆桌的记忆 .arena/rooms/research/memory/claude-opus-4-7.md ← Opus 4.7 在投研圆桌的记忆（隔离） .arena/rooms/general/memory/_profile.md ← 三家共识层（DeepSeek 派生，≤20 条） .arena/rooms/general/memory/pending-claude-opus-4-7.json ← worker 建议 Opus 采纳的候选项

三、你问的核心问题：Opus 4.7 和 Opus 4.6 共享记忆吗？

不共享。完全隔离。

Phase 3 的设计粒度是 AI_KIND + MODEL，精确到模型版本：

模型记忆文件关系
Claude Opus 4.7claude-opus-4-7.md独立个体
Claude Opus 4.6claude-opus-4-6.md独立个体
Claude Sonnet 4.6claude-sonnet-4-6.md独立个体
Gemini 3 Progemini-3-pro.md独立个体
DeepSeek V4 Prodeepseek-v4-pro.md独立个体
Codex GPT-5.2codex-gpt-5-2-codex.md独立个体
为什么这样设计？
Opus 4.7 和 Opus 4.6 是不同的模型——推理深度、回答风格、判断标准都不同。如果共享记忆，4.6 写的偏好被 4.7 读到，等于让 4.7 继承了一份"不是自己观察到的"记忆。每个模型版本是独立的思考伙伴，记忆也应独立。
潜在代价（已知，当前不修）：
升级模型时（如 Opus 4.7 → 4.8），旧模型的记忆 "被弃用"，新模型从零开始。这是当前设计的 trade-off——精确隔离 vs 平滑升级。Phase 4+ 可考虑"同家族 fallback"（找不到 4.8 时读 4.7 的记忆作为过渡）。

五、验证节奏：从第一天到第二周

第 1 天 · 首次使用

做什么：开通用圆桌，正常聊业务（不要刻意说"记住"）

看完什么：

  • 是否有 AI 的 📒 N > 0？（真自主写）
  • 点 📒 打开 .md，source 是 self 还是 explicit？
  • 内容是不是真偏好，不是临时观点？

第 2 天 · 关键测试 ⭐

做什么：开 同 scene 新通用圆桌，直接聊业务，不提任何偏好

看完什么：

  • AI 是否自然引用昨天的偏好？
  • 引用是 "你之前提过 X，所以 Y"（自然）还是 "根据记忆系统记录..."（机械）？
  • 如果完全失忆 → 跨 meeting fix 可能没生效，需要查

第 3-7 天 · 日常积累

做什么：每天用 1-2 场圆桌，别看文件

周末看：

  • 📒 N 增长是否平稳（每场 +1~3 条）？
  • 📊 _profile.md 共识条目是否稳定？
  • 抽查 5-10 条：误记率 < 10%？

第 2 周+ · 信任建立

做什么：日常用，不刻意验证

问自己：

  • AI 在不被问的时候自然带出某条偏好？
  • 看 _profile 觉得 "这就是我"？
  • "惊喜感" 出现了吗？

六、关键 Issues（我们认为你需要知道的）

6.1 自主写入率未知

Issue：E2E 验证了链路（AI 能调 memory_write），但 自然对话中的自主写入率有多高——不知道。这取决于 MEMORY PROTOCOL 引导段是否够强但不机械。如果一周后 📒 N 全是 0，说明 prompt 引导不足；如果暴涨（每场 +10），说明 AI "为记而记"。

6.2 多模型前台兼容性是债

Issue：Phase 0-3 所有 E2E 用 Claude Sonnet（避 Opus 配额 + Gemini/Codex CLI 接入 bug 未修）。Gemini 和 Codex 作为圆桌参与者是否能正常调 memory MCP——已知还没测过。当前系统在 "已接入稳定" 的 CLI 上是 production-ready；对 Gemini/Codex，要等接入 bug 修完再补测。

6.3 模型升级时记忆断层

Issue：Opus 4.7 → 4.8 升级后，新版本读不到旧版本的记忆文件（因为文件名不同）。当前按你的拍板做了精确隔离——这是正确的初始设计。但如果频繁升级模型，会一直 "从头开始"。

6.4 跨 scene 污染风险

已验证 OK：通用和投研的 memory 完全隔离（不同 directory）。但：如果以后你在通用圆桌里聊了很多投研内容，AI 在 general/ 下记了投研相关的偏好——这些不会污染 research/ 圆桌，但会留在 general 记忆里。这不算是 bug，只是通用圆桌的 "边界感" 取决于你自己怎么用。

6.5 认知固化风险

Issue：记忆系统的本意是 "越用越懂你"，但过度记住会变成 "被过去绑架"。当前防线有 3 层：(1) MEMORY PROTOCOL「不要记」条款禁止记临时结论；(2) _profile.md 淘汰规则每轮派生都会 prune；(3) 90 天未引用的记忆会被衰减（strikethrough）。但这些防线是否足够——需要你长期使用后主观判断。

6.6 pending inbox 采纳率未知

Issue：Worker 会生成 pending 候选项（"你漏记了这个"），然后 hook 注入给 AI。AI 自决采纳或拒绝。但 AI 真的会采纳吗？采纳率多少？——测试用 explicit trigger 模拟过链路，但 AI 自主决定采纳/拒绝的行为未大规模观察过。如果采纳率趋近 0，inbox 机制就只是开销，不产生价值。

八、五个核心归属问题（直接回答）

Q1: Opus 4.7 和 Opus 4.6 共享记忆吗？
不共享。完全隔离。它们是两个独立的 .md 文件。你从 Opus 4.7 升级到 4.8 时——新 4.8 读不到 4.7 的记忆（这是已知 trade-off，不是 bug）。
Q2: 同模型同版本，今天坐 Pikachu，明天坐 Charmander——记忆是同一个文件吗？
是同一个文件。Opus 4.7 的记忆文件是 claude-opus-4-7.md，不管 Opus 坐在哪个槽位，都读写这同一个文件。这就是 identity-based（AI 是主体，槽位只是 UI 标签）。
Q3: 通用圆桌和投研圆桌的 Opus 4.7——共享记忆吗？
不共享。通用圆桌的 claude-opus-4-7.md 在 general/memory/ 下；投研的在 research/memory/ 下。不同 scene 完全隔离，互不污染。你在通用圆桌里讨论投研内容产生的偏好，不会影响投研圆桌。
Q4: _profile.md 是谁写的？AI 能改吗？
_profile.md 由后台 worker 调用 DeepSeek v4-pro 派生，AI 只读不能写。Worker 每 3 轮（+ token 阈值）读三家个体记忆 → DeepSeek 找至少 2 家共现的稳定偏好 → 写入。AI 每次新 meeting 开场自动读 _profile 注入上下文。这条红线守得很死——worker 不越权写个体记忆，AI 不改共识层。
Q5: 如果我手动删了某个 .md 文件里的某条记忆——会怎样？
系统下次读写时自动从文件重新 load。没有中间缓存层。你删了一条 → AI 下次 memory_list 就看不到 → 等于那件事没发生过。这正是你最初要的"半透明"——你用文本编辑器，记忆系统用同一份文件，零中间层。

九、一句话总结

记忆现在属于 AI 个体（Opus / GPT / DeepSeek），不属于座位（Pikachu / Charmander / Squirtle）。

同一个模型换座不丢记忆 · 不同模型同座不串记忆 · 不同场景互不污染 · 你随时可以打开 .md 文件看一切。
